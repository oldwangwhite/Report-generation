# 前端功能测试问题记录（2026-07-01）

测试范围：模板管理、素材管理、报告生成、AI 正文生成、预览导出、报告记录列表。

测试材料目录：`frontend_test_upload_files`

## 1. 登录仍是前端 mock，未接真实后端鉴权（已解决）

现象：登录页明确提示“当前为前端 mock 登录，后端完成后替换为真实注册/登录接口”。

影响：当前无法真实验证账号密码、密码长度、注册、身份权限等后端鉴权逻辑，只能验证前端角色跳转和 mock token。

建议：接入真实登录接口后，补测空账号、错误密码、密码长度、普通用户访问管理员页面、管理员权限等。

## 2. 素材选择未按专业过滤

复现步骤：

1. 创建迎峰度夏检查报告。
2. 专业选择“电气”。
3. 打开“关联素材”下拉框。

实际结果：下拉框仍显示“燃料”专业的煤库存 PDF 和 Excel 台账素材。

预期结果：电气报告默认应只显示电气素材和综合素材，燃料素材不应默认出现，避免 AI 误引用错误专业依据。

严重程度：中。

## 3. 素材管理缺少解析结果查看入口

现象：素材管理列表能显示素材名称、专业、类型、文件名、大小、状态，但操作只有“停用/删除”。

影响：无法确认 Word/PDF/TXT/XLSX 是否解析成功，也无法查看提取文本、关键词、摘要、可引用片段。

建议：增加“查看详情/解析结果/关键词/引用片段/解析失败原因”等入口。

严重程度：中。

## 4. 模板详情直接展示 JSON，用户可读性差

现象：模板详情中的“模板结构”直接展示原始 JSON。

影响：管理员难以快速确认章节结构是否解析正确。

建议：改成树形章节展示，例如 1、1.1、2、2.1 这种结构，JSON 可放到调试模式或隐藏区域。

严重程度：低。

## 5. AI 正文生成卡住，无超时失败提示

复现步骤：

1. 选择“测试-迎峰度夏模板-20260701”。
2. 关联电气素材和综合素材。
3. 创建并生成大纲。
4. 修改章节标题并保存。
5. 点击“生成当前章节”。

实际结果：页面持续显示“生成中”，进度一直是 0%，正文为空。等待超过 90 秒后仍未结束。后端报告状态为 `generating`，内容列表为空。

补充验证：模型配置测试接口可正常返回 `pong`，约 1 秒返回，说明模型连接并非完全不可用。

进一步定位：后端实际保存的模型配置为 DeepSeek `deepseek-chat`，启用状态正常，测试连接成功。直接调用正文生成流式接口时，只返回 `chapterStart`，之后没有 `chunk/progress/done/error`。在后端服务代码内复现后发现异常：

```text
AttributeError: type object 'Material' has no attribute 'deleted_flag'
```

原因位置：`app/ai/material_context.py` 中 `_selected_materials` 和 `_active_materials` 查询使用了 `Material.deleted_flag == 0`，但当前 `app/entity/material.py` 的 `Material` 实体没有 `deleted_flag` 字段。

预期结果：正文生成应成功返回内容；如果失败，应在合理时间内返回失败状态和错误提示，并允许用户重试或取消。

严重程度：高。

## 6. 生成卡住后，手动保存成功但报告状态仍为生成中

复现步骤：

1. 在正文生成卡住后，手动输入正文。
2. 点击“保存当前章节”。

实际结果：页面提示“章节内容已保存”，后端该章节状态变为 `done`，内容已保存；但报告总状态仍是 `generating`。

影响：用户和报告列表都会误判该报告仍在生成中。

预期结果：如果用户手动保存章节，应同步刷新当前章节状态；如果生成任务已失败或中断，应允许状态恢复到“部分完成/待继续生成/生成失败”。

严重程度：高。

## 7. 已导出文件存在，但报告统计仍显示已导出为 0

复现步骤：

1. 在预览导出页导出 DOCX、TXT、MD。
2. 返回报告记录列表。

实际结果：报告记录中最新文件显示为 `.md`，导出记录也存在 3 个文件；但顶部统计“已导出”为 0。

预期结果：只要报告存在导出文件，统计应计入已导出数量。

严重程度：中。

## 11. 模型配置页 API 密钥存在明文暴露风险

现象：模型配置页中 API 密钥输入框可直接显示完整密钥。

影响：管理员截图、录屏或现场演示时容易泄露密钥。该密钥属于敏感信息，不应长期明文展示。

建议：默认只展示脱敏内容，例如 `sk-****59c4`；点击查看按钮后短暂显示，离开页面、刷新页面或保存后恢复隐藏。

严重程度：高。

## 12. PDF 和 XLSX 素材已能上传，但后端素材上下文暂不支持正文解析

现象：`material_coal_inventory_audit_standard.pdf` 和 `material_coal_inventory_ledger.xlsx` 可以上传并显示为启用状态。

进一步定位：后端 `app/ai/material_context.py` 当前只对 `.txt`、`.md`、`.csv`、`.docx` 做正文提取，未看到对 `.pdf` 和 `.xlsx` 的正文解析逻辑。

影响：用户以为 PDF/Excel 素材已经可被 AI 引用，但实际生成正文时可能只能使用素材名称、类型、描述，无法引用 PDF/Excel 文件正文或表格数据。

建议：如果产品承诺支持 PDF/XLSX 素材，应补充 PDF 文本提取和 Excel 表格提取；如果暂不支持，应在上传弹窗和素材状态中明确提示“已上传但不参与正文解析”。

严重程度：中。

## 13. 后端使用指定素材时未再次校验素材专业

现象：前端生成页可以选择与当前专业不一致的素材，例如电气报告中出现燃料专业素材。

进一步定位：后端构造素材上下文时，如果传入了 `materialIds`，会按 ID 取用户选定素材，没有看到按报告专业再次过滤的逻辑。

影响：即使前端后续修复过滤，接口层仍可能收到不匹配专业的素材 ID，导致 AI 引用错误专业依据。

建议：前端按专业过滤只是第一层；后端也应校验素材专业，只允许同专业素材和综合素材进入生成上下文。

严重程度：中。

## 14. 生成过程中缺少取消、重试和错误恢复机制

现象：正文生成卡住后，页面只能显示“生成中”，用户无法取消当前生成任务，也无法看到错误原因。

影响：用户不知道是否应该等待、刷新页面、重新点击生成或联系管理员，容易造成重复操作和状态混乱。

建议：生成中提供“取消生成”“重试当前章节”“查看错误”入口；后端异常时返回 `error` 事件，并把报告状态恢复为“生成失败/待继续生成”。

严重程度：高。

## 15. 未生成完整内容时允许导出，但提醒不够明显

现象：只手动保存了一个章节，其余章节未生成时，系统仍允许导出 DOCX/TXT/MD。

实际结果：导出文件中未生成章节显示“本章节暂无保存内容”。

影响：如果用户没有仔细检查预览，可能把半成品报告当作完整报告导出。

建议：导出前增加确认提示，例如“当前还有 6 个章节未生成，是否继续导出？”并在导出文件名或状态中标记“部分内容未生成”。

严重程度：中。

## 16. 上传弹窗缺少支持格式、大小限制和解析能力说明

现象：模板上传、素材上传弹窗没有明确说明支持哪些格式、最大文件大小、哪些格式能参与正文解析。

影响：用户上传 PDF/XLSX 后可能以为系统已经完整解析，但实际未必参与 AI 生成。

建议：上传弹窗中说明：

1. 模板建议使用 DOCX。
2. 素材支持 DOCX/TXT/MD/CSV，PDF/XLSX 如未实现解析需标注限制。
3. 文件大小、空文件、损坏文件应有明确错误提示。

严重程度：中。

## 17. 模板选择后缺少结构预览

现象：报告生成页选择模板后，只提示“已选择模板”，没有展示该模板包含哪些章节。

影响：用户无法在生成前确认模板是否选对，只能生成大纲后再发现问题。

建议：选择模板后展示简化章节预览，例如“1 检查概况、1.1 检查范围、2 设备运行情况”等。

严重程度：低。

## 18. 报告状态需要进一步细化

现象：当前可见状态包括“生成中”等，但实际存在“大纲已生成、部分章节已生成、生成失败、已导出但未完成”等中间状态。

## 26. 创建报告时未校验模板和素材 ID 是否真实存在

## 33. 报告类型切换后，已选模板和素材缺少自动清理

定位结果：前端报告生成页会按报告类型过滤模板下拉选项，但没有看到切换报告类型后自动清空 `templateId` 和 `materialIds` 的逻辑；同时后端创建报告又允许模板类型错配。

影响：用户先选择迎峰度夏模板，再切换成煤库存审计报告时，旧模板值可能仍留在表单或接口参数里，导致创建出错配报告。

预期结果：报告类型切换时，应清空不匹配的模板和素材，并提示“切换报告类型会清空已选模板/素材/大纲内容”。

严重程度：中。

## 34. 综合素材不会作为通用素材展示给专业报告选择

定位结果：报告生成页素材筛选逻辑只保留 `item.major` 为空或等于当前专业的素材；测试素材中 `major=综合` 的通用素材不会出现在“电气”等专业下。

影响：通用制度、通用写作规范、综合整改要求等素材无法自然参与各专业报告生成。

预期结果：素材筛选应支持“同专业 + 综合/通用素材”的组合；后端也应按同样规则校验。

严重程度：中。

## 35. 生成过程已有取消能力但页面没有取消入口

定位结果：前端 `useReportSSE` 中已有 `cancelGenerate` 方法，但报告生成页面没有使用该方法，也没有显示“取消生成”按钮。

影响：AI 生成卡住或耗时过长时，用户只能等待、刷新页面或重复点击其他操作，容易造成状态混乱。

预期结果：生成中应显示“取消生成”按钮，并在取消后恢复到可继续生成/重新生成的状态。

严重程度：中。

## 36. 权限错误和未登录错误的 HTTP 状态码不规范

复现方式：

1. 普通用户访问 `/api/admin/model-config`。
2. 不带 token 访问 `/api/reports`。

实际结果：HTTP 状态码都是 200，只在响应体中返回 `code=403` 或 `code=401`。

影响：接口自动化测试、浏览器下载、第三方调用方和网关监控会误判请求成功；前端也存在按 `40300/40100` 判断的逻辑，与后端返回的 `403/401` 不一致。

预期结果：未登录应返回 HTTP 401，无权限应返回 HTTP 403，业务响应码也应与前端约定一致。

严重程度：中。

## 37. 后端未校验年份范围

复现方式：直接创建报告，传入 `year=1800`。

实际结果：接口创建成功，报告详情 `rpt_011` 中保存了 `year=1800`。

预期结果：后端应与前端保持一致，对年份做范围校验，例如当前前端限制为 2020 到 2035。

严重程度：低到中。

## 38. 模板上传接口未限制文件类型

复现方式：将 `README_test_files.txt` 作为报告模板上传。

实际结果：接口创建模板成功，生成了 `tpl_006`，并自动套用了默认大纲。

影响：用户可能上传了错误格式文件但系统仍提示成功，后续才发现模板结构不是文件真实内容。

预期结果：模板上传应限制 DOCX，或明确标记“文件未解析，已使用默认模板结构”，并要求管理员确认。

严重程度：中。

## 39. 模板和素材状态值缺少后端枚举校验

复现方式：

1. 调用模板状态接口，将模板状态改为 `badStatus`。
2. 调用素材状态接口，将素材状态改为 `badStatus`。

实际结果：两个接口都返回成功，并保存了非法状态值。

预期结果：状态只能是 `enabled` 或 `disabled`，删除应走独立删除接口；非法状态应返回参数错误。

严重程度：中。

## 40. 系统关键配置和关键资源管理确认仍是占位页

定位结果：前端路由中 `/admin/system-config` 和 `/admin/resources` 指向 `SystemPlaceholderPage`，页面文案为“模块位置预留”；`/admin/admin-users` 已经有真实用户与角色权限页面。

影响：如果需求书要求系统关键配置、关键资源管理具备实际维护能力，目前这两个入口还没有业务闭环。

预期结果：明确这两个入口是否属于本次交付范围；如果属于，应接入真实接口和保存逻辑；如果不属于，应在验收范围中说明为预留页面。

严重程度：待需求确认。
